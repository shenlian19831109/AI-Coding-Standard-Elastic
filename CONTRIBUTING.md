# Contributing to AI Coding Standard - Elastic

We welcome contributions to the AI Coding Standard - Elastic! Your input can help us make this standard even more robust and adaptable. Here are some guidelines to help you get started.

## How to Contribute

### 1. Suggest Enhancements or Report Bugs

If you have an idea for a new feature, an improvement to an existing guideline, or you've found an issue with the standard, please open an issue on our [GitHub Issues page](https://github.com/shenlian19831109/AI-Coding-Standard-Elastic/issues).

When opening an issue, please:

- Clearly describe the enhancement or bug.
- Provide context or examples where the standard could be improved or is failing.
- Explain why the enhancement would be valuable.

### 2. Submit Pull Requests

We encourage you to contribute directly by submitting pull requests. Here's a general workflow:

1.  **Fork** the repository.
2.  **Clone** your forked repository to your local machine.
    ```bash
    git clone https://github.com/YOUR_USERNAME/AI-Coding-Standard-Elastic.git
    ```
3.  **Create a new branch** for your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    ```
    or
    ```bash
    git checkout -b bugfix/issue-description
    ```
4.  **Make your changes** and ensure they adhere to the existing structure and principles of the standard.
5.  **Test your changes** thoroughly (if applicable, e.g., for new skill files).
6.  **Commit your changes** with a clear and descriptive commit message.
    ```bash
    git commit -m "feat: Add new mode definition X" 
    # or "fix: Clarify guideline Y"
    ```
7.  **Push your branch** to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
8.  **Open a Pull Request** to the `main` branch of the original repository.

### 3. Improve Documentation or Translations

Good documentation is crucial. If you find areas where the `README.md`, `CLAUDE.md`, `GLOSSARY.md`, or any skill files can be improved, or if you'd like to contribute translations to new languages, please submit a pull request.

## Code of Conduct

We expect all contributors to adhere to our Code of Conduct. Please ensure your interactions are respectful and constructive.

Thank you for your contributions!
